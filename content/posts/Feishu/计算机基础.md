# CPU

中央处理器。负责执行程序 。

程序是由一个个操作组成的，叫做指令，指示计算机做什么。

如果是加减，这种指令，那么 CPU 会让 ALU 进行数学运算；如果是内存指令，CPU 就会和内存通信，然后读/写值。

举例：4个寄存器存临时的数据和操作数据。程序也可以用二进制表述。给 CPU 支持的所有指令分配一个 ID，

![](images/计算机基础-025.webp)

前四位表示操作码，后四位表示数据来自哪里，RAM 或者 寄存器。

另外两个寄存器，一个追踪程序执行到哪里了，这个就是指令地址寄存器；另外一个是指令寄存器。

启动计算机时候，所有寄存器都从0开始，在 RAM 假设有一个程序。

CPU 的过程：

1. 取指令阶段。

![](images/计算机基础-026.webp)

RAM 返回0地址的值，数据会被复制到指令寄存器中，弄清是什么指令，才能执行。

auto. 解码阶段

前四位操作码0010，对应 Load_A 指令，读 RAM 的地址，把对应地址的 RAM 的值放入寄存器 A。后四位是 RAM 的地址，1110，转化为十进制是14。指令由控制单元进行解码，简化地，有一个判断是否为0010的判断电路。

auto. 执行阶段

判断是否为0010的判断电路连接RAM 的 Read 使能和寄存器 A 的使能，把地址14传进去，然后数据连接四个寄存器，这个时候 寄存器 A 就获得了 RAM 中地址为14的数据了。

![](images/计算机基础-027.webp)

指令结束，然后关闭所有线路，指令地址寄存器+1，然后执行阶段结束。

不同的指令会有不同的逻辑电路进行解码，配置 CPU 内的组件来执行对应的操作。

![](images/计算机基础-028.webp)

继续

- 取指令，地址是1，所以，00011111,0001对应指令 Load_B，后四位 RAM 地址是15.

- 解码，解码对应的0001指令

- 执行，去读RAM15的地址，寄存器 B 中写入15的数据。00001110。指令+1

继续

- 取指令，10000100,1000对应指令 ADD，把两个寄存器的值加起来，和存在第二个寄存器里面，0100，01对应寄存器B，00对应第寄存器A。

- 解码，寄存器 B 的值是00001110，寄存器 A 的值是00000011。

- 执行，用 ALU 相加，获得的值放在寄存器 A 中，指令+1.

![](images/计算机基础-029.webp)

继续

- 获取指令：01001101,0100对应 Store_a，把寄存器 A 的值放入RAM。RAM 地址是13.

- 解码：

- 执行：RAM 允许读取，寄存器 A 允许写入，A 的值传给 RAM，更改RAM地址13的数据。

不断的，Fetch-Decode-Execute 的这个是时钟来负责管理的。这个过程叫做时钟速度，单位是 Hz

时钟精确的触发电信号，控制单元用这个信号递进 CPU 的操作。

第一个单芯片的 CPU 是 Intel 的4004,1971年发布的4位 CPU，时钟速度达到了740kHz。

![](images/计算机基础-030.webp)

# 指令和程序

CPU 之所以很强大，因为它是可编程的，写入不同指令，执行不同任务。它是一块硬件，但是可以被软件控制！

指令和程序都是存在同一个内存里面的，所以对于指令和程序来说，必须有一个指令，表示程序结束。便于区分指令和数据。

现代 CPU 常用两种方法来：

- 用更多位来代表指令。32位或者64位指令长度

- 可变指令长度，比如某个 CPU 用8位长度的操作码，如果看到 HALT 指令，HALT 不需要额外的数据，如果看到 JUMP 这种指令，后面肯定有一个指令位置的值的，所以就需要一个“立即值”。

比如，1971年的第一个 CPU，4004，支持46个指令，

# 高级CPU设计

 早期CPU 提速的方式，是通过减少晶体管的切换时间。但会遇到瓶颈

后来处理器厂商发明了各种新技术，让指令运行更快，也能进行更复杂的运算。

比如，现代 CPU 在硬件层面设计了除法，直接给 ALU 除法指令。这让 ALU，更大更复杂。

现代处理器还有专门的电路来处理图形操作，解码压缩视频，加密文档等。

现代处理器有上千条指令，各种复杂巧妙的电路。

更快的 CPU 带来问题，如何更快的传输数据。

数据从 CPU 搬运到 RAM 之间的线，叫做总线。

解决延迟的方法之一：

- 给 CPU 加一点 RAM，叫做缓存，cache，在 CPU 里面，一般只有几个 KB 或者几个 MB。想要的数据如果已经在缓存了，叫做缓存命中。如果缓存和 RAM 不一致了怎么办，就会有一个 dirtybit，脏位来判断。当缓存满了，CPU 还需要缓存时候，这个时候去同步，清理缓存腾出空间，先检查脏位，如果是脏的，就把数据写回 RAM

- 另一个方法，是指令流水线，

# 早期的编程方式

程序不会预先就在 RAM 里面，需要加载到内存里面。

最早的编程来自于纺织机，1801年发明的可编程纺织机，每一行的图案可以有串口纸卡决定。

![如果是高的，就勾起，如果是低，纱线不动，保留底色](images/计算机基础-034.webp)

这种纸卡的思想后来被用于统计人口信息，汇总级

![](images/计算机基础-035.webp)

后来发明了用线连接，然后来执行对应程序，但是对于复杂程序，连接线需要很久，比如第一台计算机 ENIAC，一个新程序可能接线就要3周。

所以后续人们尝试把数据和程序存在一个地方，“冯诺依曼”架构就诞生了。

冯诺依曼计算机的标志是：一个处理器（有 ALU）+数据寄存器+指令寄存器+指令地址寄存器+内存（存数据+指令）

程序和数据仍然需要以某种方式输入到计算机中，用的是穿孔打纸卡。读卡器一个个写进内存。

# 编程语言

早期的这些编程人员都需要对硬件底层有非常深厚的理解，但是在硬件底层编写代码非常复杂，需要一种更通用的方法编程。

提出需求，然后用把这个需求转化为二进制的机器码。

后面就发展出了，汇编。把汇编语言转化为机器码，比如 LOAD_A 14;越来越发展。。程序就隐藏了很多不必要的细节，可以做很多更复杂的工作。

可是，（本质还是修饰了一下机器码），汇编和机器指令是一一对应的。现在的高级语言，一行可能会转化成几十条二进制指令。1952年诞生了第一个编译器，把高级语言转化成汇编或者机器码。

![](images/计算机基础-036.webp)

1959年，学术界、政府、工业界的计算机专家们组建一个联盟，数据系统语言委员会，开发一种通用编程语言，可以在不同机器上通用，最后诞生了COBOL。为了兼容不同底层硬件，每个计算机构都需要一个 COBOL 编译器。

1980年代，诞生了 C++，Objective-C，Perl

![](images/计算机基础-040.webp)

图灵机是一个理论计算设备，有无限长的纸带，纸带可以存储符号，还有读写的头，可以读取写入纸带上的符号，还有一个状态变量，来保存当前的状态。还有一组规则，来描述机器做什么。规则是，当前状态+读写头看到的符号，决定机器做什么。结果可能是在纸带写入一个符号，也有可能是改变状态，也有可能是读写头移动一格。

# 操作系统

随着软件程序规模愈发庞大、逻辑日趋复杂，人们需要一套能自主调度计算机硬件、自动执行任务的管理程序，操作系统便应运而生。

操作系统本质上也是一段程序，但它拥有直接操控底层硬件的最高权限，负责调度硬件资源、管理其他所有应用程序，通常也是计算机开机后加载运行的第一个程序，其余所有软件都依靠操作系统启动与调度。

最早诞生的操作系统以批处理机制优化程序加载流程：在批处理系统出现前，计算机同一时段仅能运行单个程序；批处理系统可一次性载入多道任务，一道程序执行完毕后，系统会自动切换执行下一道程序，这套运行机制被称为批处理（Batch Process）。

计算机逐步普及后，硬件配置的差异化问题愈发突出：不同设备即便搭载同款 CPU，搭配的打印机等外设型号也各不相同。这给程序员带来极大负担 —— 开发时不仅要完成业务逻辑编写，还要单独适配各类外设的底层交互逻辑；而硬件厂商往往不会开放底层交互协议，进一步抬高了开发门槛。

为简化软件开发、打通软硬件交互壁垒，操作系统承担起软硬件中间媒介的角色。系统对外封装硬件细节，通过标准化 API 与设备驱动程序完成外设抽象，程序员只需统一接口即可完成 IO 交互，无需关注不同硬件的底层差异。 举个例子：程序员仅需编写打印分数的代码`print(highscore)`，操作系统便会自动接管硬件通信、打印时序、设备控制等全部底层细节。

20 世纪 50 年代，CPU 运算速度大幅提升，但外设读写速度差距悬殊，处理器时常处于空闲等待状态，大量程序因 IO 操作陷入阻塞，硬件资源严重浪费。

50 年代末期，英国曼彻斯特大学研发出全球首台超级计算机 Atlas，并配套设计调度程序最大化硬件利用率。1962 年完工的 Atlas Supervisor 是一套完整操作系统，它在传统批处理基础上新增进程调度（Scheduling） 能力，实现单颗 CPU 同时并发运行多道程序。

举个直观例子：Atlas 上运行的游戏执行打印分数指令`print(Highscore)`时，打印操作需要上千个时钟周期才能完成，速度远慢于 CPU。与其让处理器空等外设完成任务，系统会将当前程序挂起休眠，调度 CPU 去执行另一道程序。 待打印机完成输出后，会向 Atlas 系统发送完成信号，系统随即将休眠程序标记为就绪状态，等待合适时机重新分配 CPU 资源执行。

![](images/计算机基础-046.webp)

借助这套调度逻辑，Atlas 可实现并行工作：CPU 运行一道程序、打印机输出另一道程序、磁盘读取第三道程序的数据。单 CPU 依靠快速切换轮流执行多个程序、分时共享算力的技术，被称作多任务处理。

多任务并发随之带来内存资源冲突问题：每道程序运行都会占用物理内存，程序切换时不能丢失原有运行数据。对应的解决方案是为每一道程序划分独立专属内存区域。 假设整机拥有一万个物理内存地址，程序 A 占用 0–999 地址段，程序 B 占用 1000–1999 地址段；若程序申请扩容内存，由操作系统统一审核分配，扩容后的内存块可能分散在物理内存的多处位置。

实际开发中，一款程序的物理内存往往零散分布在数十个不连续区间，若让程序员直接操作物理地址，追踪管理会极其繁琐。为屏蔽内存碎片化的复杂细节，操作系统引入虚拟内存机制：所有程序都默认自身内存地址从 0 开始连续排布，真实物理地址的映射、转换工作全部由操作系统底层封装处理。

以上述程序 B 为例，即便它的物理内存分散存储，在程序自身视角中，可用地址依旧是连贯的 0–999；若程序 A 被分配两块间隔的物理内存，虚拟内存也会为它拼接出一段连续的逻辑地址空间。 依托虚拟内存，程序运行时可灵活扩容存储空间，这套能力即为动态内存分配，极大降低了多任务并发下的内存管理复杂度，为操作系统同时调度大量程序提供了充足灵活性。

![](images/计算机基础-047.webp)

给程序划分独立内存区间还附带另一重核心优势：内存隔离与内存保护。若某一个程序出现异常、随意篡改内存数据，它的操作只会限制在自身专属内存块内，不会破坏其他程序的数据，避免程序故障互相干扰。

步入 70 年代，计算机性能提升、硬件成本大幅下降，各大高校开始批量采购计算机设备，单台主机可连接数十台终端（最多支持 50 个用户同时接入）。此时系统不仅要管理多道程序，还要兼顾多名用户的资源分配；为防止单一用户独占整机算力、内存等资源，分时操作系统正式问世，系统会把 CPU 时间片均匀切分，每位用户仅分配一小部分硬件资源，即便仅 1/50 的算力，也足以支撑日常基础任务。

1969 年发布的 Multics 是业内首款分时操作系统，也是首个从设计之初就重视权限安全的系统，能够隔离不同用户的数据，阻止未授权访问；但受限于当时的硬件水平，整套系统架构极度复杂，运行需要 1MB 内存，在那个年代硬件开销过高，落地成本巨大。

基于 Multics 的开发经验，研发团队重新设计出架构轻量化的操作系统 ——Unix。Unix 整体分为两大模块：一是内核（Kernel），负责核心功能：内存管理、多任务调度、IO 外设处理等；二是配套的各类实用工具软件。 Unix 于 1971 年正式发布，后续开发者为其适配了各类编程语言编译器、文字处理工具，凭借简洁高效的架构，成为 70 至 80 年代全球最主流的操作系统。

到 80 年代，个人计算机价格下探至普通消费者可承受范围，家用 PC 配套的操作系统设计也趋于轻量化，代表产品便是微软 MS-DOS，整套系统仅占用 160KB 内存。但它缺少多任务调度与内存保护机制，一旦程序出现非法内存操作、异常崩溃，系统便会直接卡死报错。

# 计算机网络

用户可以在全球网络发送和接收信息的能力，改变了全世界。

1970年代，大部分计算机都是独立运行的，随着计算机越来越流行，数据、资源开始分享开始逐渐有用起来，

第一个计算机网络出现在1950-1960年代，在公司或者研究室内部，为了方便信息交换。

近距离构成的小型网络，Local Area Networks，LAN。局域网。可以小到是同一房间的两台机器，也可以是大到校园里的上千台机器。最成功的还是以太网 Ethernet。

基本原理是：每台设备有一个 MAC 地址，Media Access Control Address，这个唯一的地址放在头部，作为数据前缀发送到网络之中，设备监听以太网的电缆，只有是自己的 MAC 地址才处理数据。

载体传输数据的速度叫做带宽，载体有铜线，空气（对于 Wifi）。

当设备增多，共享载体可能会遇到冲突，同时想要传输数据的时候，那么可以先等待一秒，如果还冲突就等待2s，然后等待4s，16s，指数退避原则😂。

为了减少冲突，需要减少同一载体中的设备数量，载体和其中的设备总称“冲突域”。

可以通过交换机，把冲突域减小。

![](images/计算机基础-052.webp)

交换机位于两个更小网络之间，必要时在两个网络间传输数据，交换机会记录一个列表，写着 MAC 地址在哪边网络。

![](images/计算机基础-053.webp)

传输数据另一个办法就是“报文交换”，Message Switching。

从 A 传输到 B，消息会经过好几个站点，报文交换的好处就是可以用不同的 route（路由），是的通信更加可靠，容错率更好。

消息沿着路由跳转次数，叫做“跳数”（hop count）。

消息如果太大会堵塞网络，办法就是把大的报文分成很多块，叫做数据包 package，每个数据包都有目标地址。报文的具体格式有19070年代的互联网协议定义的，叫做 IP，Internet Protocal。

每台联网计算机都有 IP 地址，把数据拆分成多个小的数据包，然后通过灵活的路由进行传递，这叫分组交换。

计算机、手机等首先要连接到局域网，LAN。

广域网，先连接到一个区域性的路由器，这个路由器可能覆盖整个街区。然后连接到更大的 WAN，可能覆盖整个城市。最终达到互联网主干，互联网主干由一群超大型、带宽超高的路由器组成。

举例：想要看 b 站一个视频，首先数据包要连接到互联网主干，沿着主干 到 有对应视频文件的 YouTube 服务器，

浏览器是如何知道我们服务器的地址是多少的呢？

- 我们看的视频就是一个个数据包，而不是一整个大文件发过来。

- 数据包必须遵循 IP 协议，并且数据包大小数量也有限制。

- IP 是非常底层的协议，IP 的头部是目标地址，但是不知道是给哪个程序的（比如这个数据是给微信的？还是给钉钉的？）。

![](images/计算机基础-055.webp)

- 因此，就出现了一个 UDP，User Datagram Protocal，用户数据报协议，UDP 的头部有很多有用信息，比如端口号，每个想访问网络的程序都要向操作系统申请一个端口号，checksum 是校验和（把所有数据求和对比），检查数据是否正确。

![](images/计算机基础-056.webp)

> IP 负责把数据包送到正确的计算机，UDP 负责把数据包送到正确的程序。UDP 不提供数据修复或者数据重发的机制，也无法知道数据包是否达到目标地。有些程序不 care 这些，因为 UDP 又简单又快捷。

- 如果要保证所有数据必须到达，需要用TCP，Trasmission Control Protocal， 传输控制协议，TCP 的 Header 也存在 Data 前面，这个组合叫做 TCP/IP。也有端口号、校验和，但 TCP 有更高级的功能。

![](images/计算机基础-057.webp)

1. TCP 数据包有序号，接收方可以根据顺序把数据包排成正确顺序，哪怕顺序是乱的，TCP 协议也能排序正确。

auto. TCP 要求接收方收到数据包（packet）并且校验和检查无误后，给发送方发送一个确认码，代表收到（acknowledgement，确认码简称 ACK）。确认码的成功率和来回时间可以推测网络的拥堵程度。TCP会用这个信息调整发包数量，解决拥堵问题。

> TCP 可以处理乱序和丢失数据包，丢失了就重发😄；还可以根据拥堵情况自动调整传输率。缺点就是，确认码ACK 把数据包的数量翻了一倍。对于有些及时性要求很高的程序，比如射击游戏，这种代价就不能接受。

有了 IP 地址和端口号，我们就可以访问对应网站，但是记忆一连串数字很烦，所以有域名的出现，域名就会和 IP 地址一一对应，这个就是域名系统，Domain Name System，DNS。

比如输入了一个 bilibili.com，浏览器访问 DNS 服务器，获取到 IP 地址，DNS 查表，如果域名存在，返回对应 IP地址

# 万维网 World Wide Web

万维网在互联网上运行，互联网是传递数据的管道，各种程序都会用，传输数据最多的就是万维网，分布在全球服务器上，可以通过浏览器访问。

万维网基本单位是单个页面，超链接形成巨大的互联网络（关联式索引），用文字超链接，叫做超文本hypertext。

网页互相连接，那么每个网页需要一个唯一的地址，叫做“统一资源定位器Uniform Resource Locator”，URL。

![](images/计算机基础-060.webp)

浏览器访问网站时，首先向 DNS 服务器发起查询，把域名解析为 Web 服务器对应的 IP 地址；之后浏览器向该 IP 的80 标准网页端口发起请求，与网页服务器建立 TCP 可靠连接。

![](images/计算机基础-061.webp)

TCP 通道建立完成后，浏览器会使用 HTTP（超文本传输协议，Hypertext Transfer Protocol） 发起资源请求。 最常用的就是`GET`请求：当我们访问页面、点击网页链接时，设备会向 Web 服务器发送 GET 报文，索要对应地址的网页资源；服务器接收请求后，把网页 HTML、图片、样式等数据返回给浏览器，最后浏览器解析、渲染全部资源，将完整页面展示在屏幕上。

返回HTTP 状态码告知本次请求处理结果，比如400-499就是报错，典型的就是404，网页不存在😄。

# 计算机安全

计算机安全可以看作是保护系统和数据的：保密性，完整性和可用性。

- 保密性：只有有权限的人才能读取计算机系统和数据。

> 黑客泄露个人信息，信用卡信息，就是攻击保密性。

- 完整性：只有授权的人才能修改系统和数据。

> 黑客知道你的邮箱密码，假装你发邮件，就是攻击完整性。

- 可用性：有权限的人可以随时访问系统和数据。

> 发大量的假请求到服务器，让网站很慢或者挂掉，这就是攻击可用性。

为了实现三个目标，安全专家会从抽象层面想“敌人”可能是谁--威胁模型分析。

攻击手段又叫做：攻击矢量

最常见的：PIN，Personal Identification Number，个人识别码。

隔离，我们可以沙盒程序，

有些黑客会寻找并修复软件漏洞，让系统更安全，这就是“白帽子”。“黑帽子”就是窃取，利用，销售计算机漏洞和数据。

远程攻击一般需要攻击者利用系统漏洞。一种常见的漏洞利用叫做缓冲区溢出。
